PROJECT OVERVIEW
================================================================================

Name: G-5500 Hamlib Adaptor
Purpose: Network-connected physical adaptor providing a Hamlib interface for 
         the G-5500 Az+El rotator, primarily for Gpredict

Hardware: Wiznet Pi Pico (W5500-EVB-Pico) with RP2040 microcontroller
Language: Rust (no_std, embedded)
Framework: Embassy (async runtime for embedded systems)
License: BSD 3-clause License

Key Features:
- Async/concurrent network handling (up to 4 simultaneous HamLib clients)
- Real-time azimuth/elevation monitoring via ADC
- Relay-based rotator control (CW/CCW for Az, UP/DN for El)
- DHCP with graceful failure (device continues operating without network)
- LED status indicators (network status + client connections)
- Watchdog timer for system reliability
- Git version tracking in firmware

================================================================================
ARCHITECTURE
================================================================================

Task-Based Concurrency:
  - ethernet_task: W5500 driver runner
  - net_task: Network stack runner
  - led_blink_task: System LED status indicator
  - control_task: Rotator control (Az/El relay outputs)
  - adc_task: Position monitoring (Az/El ADC inputs)
  - listen_task: TCP socket handler (pool of 4)

Shared State Management:
  - Mutex<ThreadModeRawMutex, RefCell<T>> pattern
  - Static state for Az/El degrees, demand values, network status
  - Thread-safe access without heap allocation

Protocol Implementation:
  - HamLib rotctld protocol (partial implementation)
  - nom parser combinators for command parsing
  - Supported commands: GetInfo, GetPos, Stop, Park, SetPos, Quit, DumpState, Reset

Hardware Integration:
  - SPI0 for W5500 Ethernet (50 MHz)
  - ADC for position monitoring (10kHz sampling, 512 samples per 100ms)
  - GPIO for relay control (4 outputs) and LED status (2 outputs)
  - DMA for SPI and ADC transfers
  - Watchdog timer (8.3s timeout)

HARDWARE ARCHITECTURE
================================================================================

Microcontroller: RP2040 (Cortex-M0+)
  - Clock: 125 MHz
  - RAM: 264KB (256KB main + 8KB scratch)
  - Flash: 2MB
  - GPIO: 30 pins (28 usable)
  - ADC: 4-channel 12-bit
  - SPI: 2 instances
  - DMA: 12 channels
  - Watchdog: 8.3ms - 8.3s timeout

Pin Assignments:
  Control Outputs (GPIO):
    - GPIO 2: Azimuth CW
    - GPIO 3: Azimuth CCW
    - GPIO 4: Elevation UP
    - GPIO 5: Elevation DN

  Position Inputs (ADC):
    - GPIO 26: Azimuth position (ADC0)
    - GPIO 27: Elevation position (ADC1)

  Status LEDs:
    - GPIO 25: System LED (onboard, network status)
    - GPIO 15: Sockets LED (external, client connections)

  Ethernet (W5500 via SPI0):
    - GPIO 16: SPI MISO
    - GPIO 17: SPI CS
    - GPIO 18: SPI CLK
    - GPIO 19: SPI MOSI
    - GPIO 20: W5500 Reset
    - GPIO 21: W5500 Interrupt

Voltage Divider:
  - G-5500 outputs 0-5V
  - RP2040 ADC accepts 0-3.3V
  - 10kΩ/10kΩ divider (ratio 0.5)
  - 5V input → 2.5V at ADC

Power Budget:
  - USB Input: 5V @ 500mA
  - RP2040 Core: ~50mA
  - W5500: ~130mA
  - Relays: ~200mA (external power recommended)
  - Total: ~400mA (within USB limit)

Memory Layout:
  Flash:
    0x10000000 - 0x10000100: Boot2 (256 bytes)
    0x10000100 - 0x101E0000: Firmware (~1920KB)
    0x101E0000 - 0x10200000: Config (128KB, reserved)

  RAM:
    0x20000000 - 0x20040000: Main RAM (256KB)
    0x20040000 - 0x20041000: Scratch A (4KB, optional)
    0x20041000 - 0x20042000: Scratch B (4KB, optional)

Current Usage:
  - BSS (static data): ~126KB (48%)
  - Stack: Allocated from remaining
  - Heap: Not used (no_std)
  - Available: ~138KB

HAMLIB PROTOCOL SUPPORT
================================================================================

Implemented Commands:
  _ or \get_info          - Get device information
  p or \get_pos           - Get current azimuth and elevation
  S or \stop              - Stop rotator movement
  K or \park              - Move to park position (180°, 0°)
  P or \set_pos AZ EL     - Set target position
  q or \quit              - Close connection
  \dump_state             - Get full device diagnostics
  R or \reset             - System reset (via watchdog)

Response Format:
  - Commands return "RPRT 0\n" for success
  - Commands return "RPRT 1\n" for error
  - Position queries return "AZ\nEL\n" format
  - Info returns "Info \"<product>, firmware: <version>\"\n"
